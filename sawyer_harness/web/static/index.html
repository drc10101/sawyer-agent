Settings

Main Model

The LLM that powers the main agent. Changes take effect on the next message.

Agent Mode

Direct answers you itself, uses its own tools. Fast, cheap, good for most conversations. Orchestrator decomposes goals into subtasks, delegates to sub-agents, evaluates results. Slower but more capable for complex work. Auto starts direct, escalates to orchestration when needed.

Response Style

Controls how verbose the agent's responses are. Concise gives short answers with no tool output. Normal is balanced. Thorough explains reasoning in detail.

Max Tool Rounds

Maximum number of tool-call loops before the agent stops. Lower values prevent runaway tool usage for simple questions.

20

Show Tool Output

When enabled, tool results are streamed into the chat as they arrive. Disable for cleaner output.

Agent Rules

Custom rules that supersede runtime defaults. Rules are injected into the agent's system prompt after capabilities rules. Priority levels: P0 (critical), P1 (high), P2 (medium), P3 (low).

Add Rule







Sub-Agents

Sub-agents are specialized templates that run under your main agent. Each sub-agent has its own system prompt, personality, rules, and model settings. Create a template, then spawn a session from it to get a focused agent ready for a specific task.

SSH keys, API keys, and tokens stored locally at ~/.sawyer-harness/keys.yaml. Values are always masked in the UI. Agents respect the permission level set on each key.
